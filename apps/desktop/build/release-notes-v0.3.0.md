# Storydex v0.3.0 Windows 正式版

Storydex v0.3.0 现已发布。本版本重点升级 Agent 的意图理解、检索与记忆能力：Agent 改用 LLM 结合项目语义做意图路由，准确理解你想续写、改设定还是整理资料；续写时会自动带上章节摘要、相关旧文与 WIKI 参考，Agent 也学会了在引用旧情节前先查证，显著减少长篇写作中的前后矛盾。桌面版应用内更新支持差分下载，升级新版本不再需要重新下载完整安装包。同时改进 WIKI 图谱稳定性、上下文配置能力，并强化项目文件安全边界。

建议所有用户升级到此版本。

## 新增功能

- 新增应用内差分更新。在「系统设置 → 更新与关于」检查并下载新版本时，只下载与已安装版本有差异的部分，不再重新下载完整安装包，显著节省流量和等待时间。首次通过应用内更新升级时仍会下载完整安装包，之后的每次升级自动使用增量下载。
- 新增智能意图路由。Agent 判断你这轮想做什么（续写正文、设计角色、维护世界书、写剧本大纲、整理 WIKI、整理项目目录等）不再依赖关键词匹配，而是由 LLM 结合项目技能注册表与上一轮对话做结构化分类：「然后呢」「继续」这类省略式请求也能正确延续上一轮意图；识别出的意图会连同产出落点（如角色设定 → `.storydex/characters/`）和匹配技能一起注入执行上下文，Agent 清楚这轮产出该写到哪里。项目里自定义的新技能类型会自动加入可识别范围。LLM 不可用或超时会自动回退关键词模式，功能不中断。
- 新增 Agent 项目检索工具。Agent 现在可以对章节正文、角色、世界书和记忆做相关性排序的全文检索（针对中文优化），也可以直接查询 WIKI 知识图谱中的实体、关系与伏笔。续写涉及较早剧情时，Agent 会先查证原文再落笔，查不到的细节不再凭空编造。
- 新增滚动章节摘要。Agent 每轮续写会顺带维护一份 150-300 字的本章摘要，下一轮生成时自动注入最近章节的剧情脉络，弥补「最近几段正文」与「人物设定」之间缺失的中程记忆。
- 新增相关旧文段落召回。生成上下文会自动检索与当前活跃角色相关的旧文摘录（如几十章前的道具、地名、伏笔），作为续写参考。
- 新增 WIKI 参考条目注入。当前场景涉及的角色/设定若已有 WIKI 条目，其摘要会连同置信度一起注入上下文；待审阅条目不会注入，避免未确认的推断影响正文。
- 新增「上下文窗口（tokens）」配置项。在 LLM 配置面板中可按所用模型填写真实上下文窗口，Agent 的自动压缩阈值随之生效——使用小窗口模型时不再因阈值失真而爆上下文。

## 改进

- 界面视觉全面收敛：清理早期遗留的「药丸按钮 + 大圆角卡片 + 悬浮光晕」风格，统一为贴近 VS Code 的紧凑工作台质感——全局圆角收敛到 4/6px 两档（仅设置窗口 8px），按钮、输入框、弹层、标签全面锐化；悬停不再位移发光，顶栏与弹窗遮罩去除毛玻璃模糊，弹层投影收敛为贴边短投影；圆形语义元素（加载环、进度环、头像、开关、计数徽章）保持全圆。同时保留 Storydex 的暖色强调与五套主题个性。
- 续写上下文全面重排：最近正文片段紧随预设注入，不再被记忆块挤占预算；上下文总预算从 7600 字符提升到 10000 字符，尾部信息不再被静默丢弃。
- WIKI 关系边容量从 300 提升到 1200，超出上限时按「结构关系优先、权重降序」智能淘汰，不再按顺序硬截断；角色共现关系去除了方向重复。
- Agent 的文件读写与命令执行显式绑定当前小说项目目录，不再依赖进程级目录切换，多项目、多会话场景更稳定。
- 全文检索索引增量更新：只重建有变动的文件，已删除文件自动清出索引。

## 修复

- 修复续写时章节目录反复产生同名空目录的问题（如同时出现「第1章 苏家少年」和空的「第一章 苏家少年」）。根因是章节规范名用中文数字、而默认模板与实际落盘用阿拉伯数字，目录归一重命名与写入互相打架。现在章节命名统一为阿拉伯数字（第1章、第2章），异体命名的写入路径会自动归并到现有章节目录，历史遗留的同章号空目录会在下次打开或生成时自动清理；章号唯一的空章节目录（例如你手动新建的待写章节）不受影响。
- 修复 WIKI 章节条目在插入、删除或重命名章节后互相错位覆盖的问题。章节条目现在使用稳定标识，旧图谱会在下次打开或同步时自动完成迁移，无需手动处理。
- 修复预设编译失败被静默忽略的问题。编译出错时对话流会显示「⚠ 预设编译失败（已回退为原文摘要）」，预设状态工具同步报告编译健康度。
- 修复中文全文检索基本失效的问题（索引改用中文二元分词）。
- 修复 WIKI 同步在异常数据下可能报错中断的问题。

## 安全

- Agent 的写入操作新增工作区硬边界：目标路径不在当前小说项目内时一律拒绝，任何权限模式（包括完全访问）下都不例外。默认权限保持「完全访问」，日常使用体验不变。

## 升级说明

- 从 v0.1.x 升级时，请下载新版安装包直接覆盖安装；本版本起即可在「系统设置 → 更新与关于」中检查并增量更新后续版本。
- 从 v0.2.x 升级时，可直接在应用内检查更新并安装。
- WIKI 图谱数据结构升级到 v4：首次读取或增量同步时自动全量重建，过程无需人工干预。
- 全文检索索引位于项目 `.storydex/.cache/` 目录，首次使用时自动构建，可随时删除，下次会自动重建。
- 如需关闭自动旧文召回，可在项目 `.storydex/config/feature-flags.json` 中将 `CONTEXT_PIPELINE_FTS5` 设为 `false`。
- Storydex 不会主动删除用户项目；升级前仍建议确认重要写作项目已经备份或提交到版本控制。
